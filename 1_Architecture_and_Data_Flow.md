# Архитектура и Пайплайн данных (Data Flow)

## Описание системы
Система представляет собой автоматизированный конвейер (ETL + LLM Evaluation) для сбора неразмеченных русскоязычных научных статей, синтеза QA-датасетов на их основе и бенчмаркинга локальных языковых моделей (LLM) с использованием подхода "LLM-as-a-Judge".

## Технологический стек
*   **Сбор данных:** `sickle` (OAI-PMH), `requests`, `BeautifulSoup` (HTML-парсинг).
*   **Обработка документов:** `pymupdf4llm`, `pymupdf-layout` (OCR и PDF to Markdown).
*   **Синтез и Оценка (LLM API):** `google-genai` (Gemini 3 Flash Preview, Gemini 3.1 Flash Lite).
*   **Inference локальных моделей:** `transformers`, `bitsandbytes`, `torch` (4-bit Quantization, NF4).

## Архитектура конвейера (Pipeline Phases)

### Фаза 1: Data Acquisition (`article_retriever.py`)
1. **Подключение к OAI-PMH:** Запрос метаданных с CyberLeninka, начиная со случайной даты (рандомизация выборки).
2. **Фильтрация (Strict Mode):** Проверка наличия кириллицы в заголовке и парсинг `<meta name="keywords">` на странице статьи для жесткой классификации по 5 рубрикам (Математика, Физика, Медицина, Философия, Право).
3. **Скачивание:** Сохранение `.pdf` файлов.
4. **Конвертация:** Преобразование PDF в `.md` с извлечением текста и таблиц с помощью `pymupdf4llm`.

### Фаза 2: Dataset Synthesis (`dataset_getter.py`)
1. **Чтение контекста:** Загрузка `.md` файлов.
2. **Prompting:** Отправка контекста в `gemini-3-flash-preview` с промптом на генерацию 4 вопросов разной сложности и эталонных ответов (Ground Truth).
3. **Парсинг:** Очистка ответа от Markdown-тегов (```json) и сохранение в иерархический JSONL-подобный массив. Текст статьи сохраняется один раз для экономии места.

### Фаза 3: Model Inference (`models_inference.py` - Part 1)
1. **Загрузка модели:** Инициализация модели, загрузка и применение квантования (для некоторых прогонов)
2. **Генерация:** Подача контекста (обрезанного для некоторых прогонов) и вопроса.
3. **Постобработка (Anti-CoT):** Удаление тегов размышлений `<think>...</think>` через регулярные выражения для получения чистого ответа.

### Фаза 4: Evaluation / LLM Judge (`models_inference.py` - Part 2)
1. **Оценка:** Отправка вопроса, эталонного ответа и ответа локальной модели в `gemini-3.1-flash-lite-preview`.
2. **Скорринг:** Получение оценки от 1 до 5 и текстового обоснования (reason).

## Структура хранения данных
Основной продукт системы — JSON файл с ответами тестируемой модели:
`Контекст + QA` -> `Ответ Модели` -> `Оценка LLM As A Judge` -> **Итоговый Benchmark JSON**.
