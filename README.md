# benchmarkLLM: Автоматизированный пайплайн оценки языковых моделей на научных статьях

**benchmarkLLM** — это end-to-end система для сбора, обработки и тестирования больших и малых языковых моделей (LLM/SLM) на сложных русскоязычных текстах. 

Проект полностью автоматизирует процесс: от парсинга свежих научных статей с КиберЛенинки до синтеза QA-датасетов и выставления независимых оценок с помощью подхода **LLM-as-a-Judge**.

## Возможности проекта

*   ** Автоматический сбор данных (Zero-Shot Data Collection):** Умный парсер, работающий через протокол OAI-PMH. Фильтрует статьи по строгим тегам (Математика, Физика, Медицина, Философия, Право), извлекает PDF и конвертирует их в чистый Markdown (с помощью `pymupdf4llm`).
*   ** Синтез датасетов (GenAI):** Использование мощных моделей (Gemini 3 Flash / 3.1 Flash Lite) для автоматической генерации сложных вопросов и эталонных ответов (Ground Truth) на основе контекста статьи.
*   ** Оценка локальных моделей (LLM Judge):** Инференс открытых моделей (например, Qwen, Llama) в 4-битном квантовании (NF4) с последующей оценкой их ответов "Судьей" (Gemini) по шкале от 1 до 5 с текстовым обоснованием.
*   ** Отказоустойчивость:** Встроенная система *Resume* (продолжение работы после сбоев) и умная обработка лимитов API (Rate Limits 429/503).
*   ** Anti-CoT (Chain-of-Thought):** Интеллектуальная очистка ответов современных моделей от тегов размышлений (`<think>...</think>`) для чистого бенчмаркинга.

## Структура проекта (Скрипты)

Пайплайн разделен на независимые модули, которые можно запускать последовательно:

1.  `article_retriever.py` — Сборщик PDF-статей и конвертер в `.md` формат.
2.  `dataset_getter.py` — Синтезатор QA-датасета (использует Google Gemini API для генерации вопросов).
3.  `models_inference.py` *(Part 1)* — Инференс тестируемой модели (HuggingFace, 4-bit) на собранном датасете.
4.  `models_inference.py` *(Part 2)* — Оценка сгенерированных ответов с помощью LLM-Judge (Gemini API).
